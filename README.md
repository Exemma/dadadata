# dadadata

Набор утилит на Python для работы с [DaData API](https://dadata.ru/) через официальный клиент [dadata-py](https://github.com/hflabs/dadata-py): баланс, суточная статистика и **стандартизация адресов** (Cleaner API). Два варианта графического приложения (классический **Tkinter** и **CustomTkinter**) дополнительно умеют **геокодировать адреса через HTTP Геокодер Яндекса** и смотреть **лимиты в API кабинете Яндекса**. Общая бизнес-логика вынесена в модуль [`address_cleaner_core.py`](address_cleaner_core.py), чтобы не дублировать код между интерфейсами.

Выход в интернет через **локальный SOCKS5** (Tor, Tor Browser, корпоративный прокси) поддерживается и в GUI, и в консольном скрипте баланса.

## Требования

- Python 3.9+
- Зависимости из `requirements.txt`

```bash
pip install -r requirements.txt
```

Пакеты: `dadata`, `httpx[socks]`, `openpyxl` (экспорт результатов в `.xlsx`), `customtkinter` (альтернативный GUI).

## Структура проекта

| Файл | Назначение |

## Возможности GUI (Tkinter и CustomTkinter)

Поведение совпадает; отличается только внешний вид и стек виджетов.

- **Стандартизировать** — построчная нормализация адресов через `clean("address", …)`; сокращённый результат справа, разворачиваемый полный вывод снизу. Горячая клавиша: **Ctrl+Enter**.
- **Получить координаты** — запросы к `https://geocode-maps.yandex.ru/v1/` (актуальный путь для ключей из нового кабинета; между запросами есть небольшая пауза, чтобы снизить риск лимитов).
- **Показать баланс** / **Показать статистику** — обёртки над `get_balance()` и `get_daily_stats()` DaData; статистика открывается в окне с человекочитаемым текстом и кнопкой «Копировать JSON».
- **Лимиты Яндекс (кабинет)** — запросы к API кабинета разработчика (`api-developer.tech.yandex.net`) по заголовку **X-Auth-Key**; в окне отображаются проекты, сервисы и суточные лимиты.
- **В Excel (DaData)** / **В Excel (Яндекс)** — сохранение последнего успешного прогона в `.xlsx` (листы с понятными заголовками; для DaData дополнительно колонки: индекс, регион, город, улица, дом и т.д.).
- **Настроить прокси** + галка **Через SOCKS5** — хост, порт, опционально логин/пароль SOCKS5; при включённой галке перед запросами выставляются `ALL_PROXY` и `HTTPS_PROXY`.

Ключи API вводятся в полях (маскирование звёздочками). Блок полей можно свернуть кнопкой «Свернуть поля ключей».

## Хранение настроек

Файл с сохранёнными ключами и параметрами SOCKS лежит **в профиле пользователя**, а не рядом с кодом — чтобы при копировании папки проекта чужие секреты не уезжали вместе с файлами.

- **Windows:** `%LOCALAPPDATA%\DaDataAddressCleaner\api_credentials.json`
- **Linux и прочие Unix:** `$XDG_CONFIG_HOME/dadadata-address-cleaner/api_credentials.json` или `~/.config/dadadata-address-cleaner/api_credentials.json`

Для файла по возможности выставляются ограниченные права (`chmod 600` там, где это поддерживается).

## Переменные окружения

Для **GUI** они **не обязательны**: ключи можно ввести в окне или подтянуться из сохранённого профиля (см. выше). Переменные удобны для автоматизации, CI или чтобы при старте подставить значения в поля: если переменная **задана** в окружении (в том числе пустой строкой), её значение имеет приоритет над сохранённым в файле.

Для **`check_dadata_balance.py`** без оболочки нужен как минимум `DADATA_TOKEN` (или передайте ключ иначе — см. раздел «Запуск»).

| Переменная | Описание |
|------------|----------|
| `DADATA_TOKEN` | API-ключ DaData |
| `DADATA_SECRET` | Секретный ключ (нужен для Cleaner и части методов) |
| `DADATA_SOCKS_PROXY` | URL SOCKS5; по умолчанию в коде `socks5://127.0.0.1:9150` (типичный порт Tor Browser). Подхватывается при старте и может заполнить поля диалога «Настроить прокси» |
| `DADATA_USE_PROXY` | Если задано значение вроде `1` / `true` / `yes` / `on`, при старте GUI включена галка «Через SOCKS5». Значения `0`, `false`, `no`, `off` — явно выключают |
| `YANDEX_GEOCODER_API_KEY` | Ключ HTTP Геокодера Яндекса (продукт вроде «JavaScript API и HTTP Геокодер») |
| `YANDEX_DEVELOPER_AUTH_KEY` | X-Auth-Key из [API кабинета Яндекса](https://developer.tech.yandex.ru/) (не путать с ключом геокодера) |

В GUI при закрытии окна ключи DaData и Яндекса перезаписываются в профильный JSON (и в версии на CustomTkinter).

Консольный `check_dadata_balance.py` при старте вызывает `apply_socks_proxy_from_env()` из `dadata_proxy.py`, то есть **всегда** настраивает прокси из `DADATA_SOCKS_PROXY` (или значение по умолчанию). Для прямого доступа без SOCKS задайте другой способ запуска или измените переменную/код под свой сценарий.

## Запуск

**Баланс (консоль):**

```powershell
$env:DADATA_TOKEN="ваш_ключ"
# при необходимости:
# $env:DADATA_SECRET="ваш_секрет"
# другой прокси: $env:DADATA_SOCKS_PROXY="socks5://127.0.0.1:9050"
python check_dadata_balance.py
```

**Стандартизация и геокодер (GUI, Tkinter):**

```powershell
python address_cleaner_gui.py
```

**Тот же функционал, интерфейс CustomTkinter:**

```powershell
python address_cleaner_gui_ctk.py
```

Ключи укажите в интерфейсе (или они подгрузятся из профиля после прошлого сеанса). При желании перед запуском можно выставить `DADATA_TOKEN`, `DADATA_SECRET` и остальные переменные из таблицы выше — они подставятся в поля и перекроют сохранённые значения.
